## 二、已完成的研究工作及成果

> 本部分对已完成工作进行详细介绍和研究成果分析。

### 2.1 研究背景与问题定义

- 2.1.1 水下无线传感器网络（UWSN）的应用场景与挑战
  - 水声信道的高衰减与高延迟特性
  - 节点能量受限与补充困难问题
- 2.1.2 现有协议的局限性
  - 传统地面 WSN 协议在水下环境的局限性
  - LEACH：基于概率轮换，不感知节点剩余能量，能耗不均
  - QELAR：Q-learning 奖励函数未充分考虑多维指标（延迟、链路质量）
- 2.1.3 本文研究目标
  - 设计一种融合多维度奖励的强化学习分簇路由协议（EC-QL），延长网络寿命，均衡能耗，并控制端到端延迟

### 2.2 仿真平台架构

- 2.2.1 平台设计原则（模块化、可扩展）
- 2.2.2 目录结构说明
  - `Core_Simulation/`：仿真主循环与参数配置
  - `Network_Layer/`：路由算法与奖励计算
  - `Physical_Layer/`：水声信道物理模型
  - `Agents/`：结果可视化

### 2.3 物理层水声信道模型与仿真实验设计

- 2.3.1 Urick 传输损耗（TL）模型
  - 公式推导：TL = k * 10*log10(d) + alpha * d * 1e-3
  - 扩散系数 k=1.5（复杂海洋环境）
- 2.3.2 Thorp 吸收系数模型
  - 频率相关的吸收系数 alpha(f) 计算
- 2.3.3 能耗模型
  - 发送能耗：E_tx = L * E_elec + P_amp * d^2 * (L/R)
  - 接收能耗、空闲能耗参数设定
- 2.3.4 信道模型验证图（Channel_Model_Curves.png 分析）
  - 不同频率下吸收系数曲线
  - 不同频率下传输损耗随距离变化曲线
- 2.3.5 仿真实验参数设置
  - 网络规模：100 节点，500×500×500 m³ 三维空间
  - 初始能量：1.0 J/节点，Sink 位于 (250, 250, 500)
  - 通信半径：325 m，最大仿真轮次：2000 轮
  - Q-learning 参数：学习率 0.1，折扣因子 0.9，epsilon=0.2
- 2.3.6 评价指标定义
  - 网络寿命：首个节点死亡轮次（FND, First Node Dead）
  - 网络总剩余能量随轮次变化曲线
  - 端到端传输延迟
  - 鲁棒性指标（FND 随外部参数变化的灵敏度）

### 2.4 对比协议实现

- 2.4.1 LEACH 协议实现
  - 基于概率 P_leach=0.05 的轮换簇头选举机制
  - 单跳直传 Sink 的路由策略
- 2.4.2 QELAR 协议实现
  - 水声网络专用 Q-value 更新规则
  - 基于 Q 值的下一跳选择逻辑（Route_To_Sink_QELAR.m）

### 2.5 本文算法 EC-QL 设计与实现

- 2.5.1 算法整体框架
  - 两层 Q-learning：分簇决策层 + 骨干路由层
  - epsilon-greedy 探索策略（epsilon=0.2）
- 2.5.2 基于 Q-Learning 的分簇选举
  - 状态空间定义：节点能量百分比离散化（18 个状态）
  - 动作空间：{簇头 CH, 成员 Member}
  - 关联逻辑：成员节点就近关联最近簇头
  - 初始拓扑结构图（Initial_Topology.png）
  - 典型轮次路由拓扑图（Final_Highlighted_Routing.png）：辅助说明簇头分布与多跳路由路径
- 2.5.3 多维度奖励函数设计（核心创新点）
  - 能量奖励 R_E：剩余能量比的四次方，鼓励保护高能量节点
  - 进度奖励 R_D：到 Sink 距离缩减比，保证数据"向前传"
  - 链路质量奖励 R_L：基于 Urick 模型计算 SNR，奖励高质量链路
  - 传输延迟惩罚 R_Delay：基于声速 1500 m/s 计算传播延迟并归一化
  - 非线性距离惩罚 R_cost：四次方归一化惩罚，抑制长距离跳跃
  - 动态权重机制：随节点能量衰减，自动提升能量权重比例
- 2.5.4 骨干路由逻辑（Route_To_Sink.m）
  - 多跳路径规划（簇头 → 中继节点 → Sink）
  - Q 表更新与收敛机制

## 四、存在的困难及解决方案

### 4.1 存在的困难

1. **超参数调优复杂度高**：EC-QL 涉及多个相互耦合的超参数（学习率、折扣因子、探索率、奖励权重等），参数空间庞大，网格搜索计算代价高，难以保证在有限时间内找到全局最优配置；
2. **仿真结果解释难度大**：多维度奖励函数的各分量对最终性能的贡献难以精确解耦，实验结果的成因分析容易流于定性描述，缺乏严格的消融实验支撑；
3. **水声信道模型的理想化限制**：当前仿真采用 Urick/Thorp 经验模型，未充分考虑多径传播、时变信道等实际水下声学效应，可能与真实部署场景存在偏差；

### 4.2 解决方案

1. 针对超参数调优，采用随机搜索与局部精细调优相结合的策略，重点围绕初步实验表现较好的参数范围进行细化扫描，同时固定部分敏感度低的参数以缩减搜索空间；
2. 针对结果解释，设计消融实验，分别测试移除单一奖励维度（如去掉 $R_{Delay}$ 或 $R_{cost}$）后的算法性能变化，从而量化各分量的贡献，为论文提供更有说服力的分析依据；
3. 针对信道模型局限性，在论文相关章节中明确说明仿真假设范围，并在未来工作部分指出引入更精确信道模型的改进方向，将其作为研究局限性坦诚处理；
4. 针对并行压力，制定详细的周计划，采用"写完一部分提交审阅、同时推进下一部分实验"的流水线方式，确保研究与写作互不阻塞。
